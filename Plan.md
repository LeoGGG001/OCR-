## 项目实现计划：古籍竖排双行夹注 OCR 识别工具

#### 1. 项目概述

本项目旨在开发一个专门针对中文古籍扫描版的 OCR 工具。核心挑战在于处理**竖排左行**、复杂的**双行小字夹注（Interlinear Annotations）**逻辑，以及提供可选的双页书脊分割功能。

#### 2. 技术栈选型

- **核心开发语言**：Python 3.8+
- **图像处理**：OpenCV (用于去噪、倾斜校正、书脊裁切)
- **OCR 引擎**：PaddleOCR (PP-OCRv4，专注于中文竖排与密集文字识别)
- **后端框架**：FastAPI 或 Flask (用于提供 OCR 接口、处理前端上传的图片)
- **前端界面**：HTML5 + CSS3 + JavaScript (可视化交互、结果展示)
- **版式分析**：基于坐标几何逻辑的自定义 Python 排序算法

#### 3. 核心算法逻辑 (Phase-by-Phase)

##### 第一阶段：核心 OCR 与数据获取 (Core Detection)
- 暂时跳过复杂的预处理，直接输入单页/清晰的古籍图片。
- 调用 PaddleOCR 获取每个文字块的 `Box` (四角坐标) 和 `Text` (内容)。
- **关键**：仅提取坐标和文字内容，摒弃引擎默认的阅读顺序。

##### 第二阶段：版式分析与排序算法 (Layout & Sorting) —— **项目的核心灵魂**
- **列划分 (Column Clustering)**：根据 `Box` 的横坐标 X 轴进行聚类，将页面从右到左划分为若干垂直列。
- **大小字判定**：计算每一列文字块的平均宽度 $W_{avg}$。宽度 $\approx W_{avg}$ 判定为大字正文；宽度 $\approx 0.5 \times W_{avg}$ 判定为小字注释。
- **夹注排序逻辑**：
  - 从右向左遍历列，从上到下读取框。
  - **排序模型**：`右上小字 -> 左上小字 -> 中间大字 -> 右下小字 -> 左下小字`。
  - 将打碎的文字块按逻辑重组为连贯的字符串。

##### 第三阶段：预处理与书脊分割 (Pre-processing & Splitting) —— **可选/交互式**
- **图像倾斜校正**：通过 OpenCV 霍夫变换将倾斜的扫描件拉平（基础功能）。
- **书脊分割（按需触发）**：
  - 后端：利用垂直投影算法初步猜测中缝位置。
  - 前端交互：向用户展示参考线，用户可确认或手动拖拽调整中线。
  - 根据确认的中线将图片裁分为左右两页，分别送入 OCR 管道。

##### 第四阶段：Web 前端 UI 与交互 (Web UI)
- **布局设计**：左右分栏结构。
- **左侧工作区**：Canvas 图片渲染。支持放大缩小、显示/隐藏识别框（Bounding Box）、拖拽书脊分割线。
- **右侧结果区**：可编辑的多行文本框（Textarea），实时展示 OCR 排序后的古文，支持用户人工校对和一键复制/导出 Txt。

#### 4. 实施路线图 (Step-by-Step Milestones)

- [ ] **M1: 跑通底层基础 (Python)**
  - 安装 PaddleOCR 和 OpenCV。
  - 编写一个 Python 脚本，能够读取一张单页古籍图，输出所有文字框的坐标和内容字典。

- [ ] **M2: 攻克核心排序算法 (Python) - MVP阶段**
  - 实现“第三阶段”中的版式分析算法。
  - 能够正确将带有双行夹注的单页测试图，按正确的阅读顺序输出到终端 (Console)。

- [ ] **M3: 搭建 Web 后端接口 (Python/FastAPI)**
  - 引入 FastAPI/Flask。
  - 编写 `/upload` 接口，接收图片并返回 M2 中的 OCR 坐标和排好序的文本数据 (JSON 格式)。

- [ ] **M4: 开发 HTML 前端界面 (HTML/JS)**
  - 编写前端页面，实现图片上传。
  - 在网页端利用 `<canvas>` 绘制原图，并根据后端返回的 JSON 数据在图上画出带有序号的半透明红色框。
  - 在右侧文本框展示最终文字。

- [ ] **M5: 加入书脊分割功能 (全栈)**
  - 前端增加“是否为双页扫描”的勾选框。
  - 后端增加中缝计算逻辑。
  - 前端 Canvas 支持显示并在图像中央绘制可拖拽的“裁切参考线”。

- [ ] **M6: 进阶与优化**
  - 解决粘连字问题、增强二值化处理。
  - (可选) 接入 LLM (如 GPT/Gemini) API，在右侧面板新增一个“AI 点校/断句”按钮。

#### 5. 待解决难点
- **异体字转换**：部分罕见字 OCR 可能识别为繁体、简体或错字，需考虑建立后处理替换词典。
- **粘连字/印章干扰**：古籍中文字过密或带有藏书印章时，可能会干扰框的检测，需在 OpenCV 预处理阶段进行阈值调试。
