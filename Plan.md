## 项目实现计划：古籍竖排双行夹注 OCR 识别工具

#### 1. 项目概述

本项目旨在开发一个专门针对中文古籍扫描版的 OCR 工具。核心挑战在于处理**竖排左行**、**双页分割**以及最复杂的**双行小字夹注（Interlinear Annotations）**逻辑。

#### 2. 技术栈选型

- **开发语言**：Python 3.8+
- **图像处理**：OpenCV (用于书脊分割、去噪、倾斜校正)
- **OCR** **引擎**：PaddleOCR (其 PP-OCRv4 模型在中文竖排识别上表现优秀)
- **版式分析**：基于坐标几何逻辑的自定义排序算法
- **UI** **界面**：PyQt6 或 Streamlit (前期推荐 Streamlit 用于快速原型展示，后期用 PyQt 做桌面端)

#### 3. 核心算法逻辑 (Phase-by-Phase)

##### 第一阶段：图像预处理 (Preprocessing)

- **自动中缝检测**：利用垂直投影算法检测图像中央的空白或黑色垂直条，自动将双页扫描件切割为左右两单页。
- **倾斜校正 (Deskewing)**：利用霍夫变换检测文字行倾斜角度并校正，确保文字框的坐标（X, Y）准确。
- **二值化****与去噪**：提升 OCR 识别的清晰度。

##### 第二阶段：OCR 识别与原始数据获取 (Detection & Recognition)

- 调用 PaddleOCR 获取每个文字块的 `Box` (四角坐标) 和 `Text` (内容)。
- **关键输出**：保留原始坐标数据，不直接使用 OCR 引擎自带的从上到下排序结果。

##### 第三阶段：版式分析算法 (The Layout Logic) —— **核心逻辑**

- **列划分 (Column** **Clustering****)**：根据 `Box` 的横坐标 X 轴进行聚类，将页面分为若干垂直列。
- **双行小字识别**：
  - 计算每一列中文字块的平均宽度 $W_{avg}$。
  - 判定宽度 $\approx W_{avg}$ 的为“大字正文”。
  - 判定宽度 $\approx 0.5 \times W_{avg}$ 的为“小字注释”。
- **夹注排序逻辑**：
  - 从右向左遍历列。
  - 在一列中，按 Y 轴（从上到下）读取。
  - 若遇到小字，寻找其左侧或右侧是否有并行的另一个小字块。
  - **排序模型**：`右侧小字块(上) -> 左侧小字块(上) -> 中间大字 -> 右侧小字块(下) -> 左侧小字块(下)`。
  - 合并为连续的字符串。

##### 第四阶段：UI 界面与交互 (Interface)

- **左侧：** 原始图像展示，并在识别出的文字上覆盖带编号的半透明框。
- **右侧：** 实时编辑框，展示生成的 Txt 文本。
- **交互：** 允许用户手动微调“书脊位置”或勾选“是否有夹注”。

#### 4. 实施路线图 (Milestones)

- [ ] **M1: 基础环境搭建** (安装 PaddleOCR, OpenCV)
- [ ] **M2: 书脊分割脚本** (实现 A.4 规则)
- [ ] **M3: 坐标排序原型算法** (实现 A.3 双行夹注逻辑)
- [ ] **M4: 完整 Pipeline 集成** (图片入，Txt 出)
- [ ] **M5:** **GUI** **界面开发** (增加可视化校对功能)
- [ ] **M6: (进阶)** **LLM** **后处理** (接入 API 进行古文纠错与断句)

#### 5. 待解决难点

- **粘连字识别**：古籍中文字有时靠得太近，导致 OCR 将两行识别为一个框。
- **异体字转换**：部分罕见字需要映射回标准繁体或简体。
